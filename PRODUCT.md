# Product

## Register

product

## Users

Students, graduate students, teachers, international students, researchers, course-report presenters, thesis-defense candidates, and business presenters who need a polished deck quickly but still want to control the storyline and visual direction.

They arrive with a topic, pasted text, or uploaded material. Their main context is deadline-driven: they want the system to understand the source, ask only the necessary questions, generate a strong outline, offer professional visual directions, and export files they can directly use or edit.

## Product Purpose

This is an international AI PPT SaaS that converts user input into a reviewed `OutlineDecision`, three premium `VisualDirection` choices, one canonical `SlideDeck JSON`, and then both editable PPTX and HyperFrames-style dynamic HTML from that same deck contract.

Success means the user feels the product actually thinks before designing: it parses source material, builds a content-aware outline, proposes distinct visual directions, resolves content-serving imagery, and exports a presentation that feels personally tailored rather than like text swapped into a few templates.

## Brand Personality

Premium, calm, intelligent.

The product should feel like a careful presentation strategist: concise, confident, Chinese-first in the local workflow, internationally capable, and quietly high-end rather than flashy.

## Anti-references

Do not look like a generic template gallery, a data-heavy dashboard, a prompt playground, or a brittle demo that only swaps text into a few fixed templates.

Avoid exposing raw prompts, provider errors, uploaded source dumps, implementation logs, or excessive technical details to ordinary users. Avoid visual clutter, over-rounded glass cards, low-contrast muted text, and generic AI SaaS boilerplate layouts that make every step feel like the same card repeated.

## Design Principles

1. Guide one decision at a time: the conversation should reveal the next question only after the current one is useful.
2. Show thinking, not machinery: make outline quality, visual rationale, image rationale, and export readiness visible without exposing implementation noise.
3. Preserve the canonical pipeline: PPTX and HyperFrames HTML must both come from the same `SlideDeck JSON`.
4. Make choice feel expert: visual directions should be distinct editorial recommendations, not a template picker.
5. Keep the interface quieter than the output: the app should stay focused so the user’s future PPT feels like the hero.

## Legal Low-Cost Agent Architecture

The product must be treated as intelligent workflow software, not as a hidden reseller of consumer web subscriptions.

Supported legal cost architectures:

1. **User-owned API keys / BYOK**: users configure their own OpenAI, Claude-compatible, Gemini, DeepSeek, Qwen, Kimi, OpenRouter, Groq, local OpenAI-compatible, or Ollama providers. The product sells workflow, prompt assets, generation contracts, QA, and export tooling; model bills stay with the user.
2. **Hybrid model routing**: cheap/local models handle extraction, cleanup, table normalization, and first drafts; mid-tier models handle outline/story/visual direction; strong models are reserved for final polish, consistency, high-stakes QA, or enterprise work.
3. **Human-in-the-loop prompt workspace**: the product can generate copyable prompt packs for the user to paste manually into their own ChatGPT/Claude/Gemini web membership, then paste results back into the workflow. This is human collaboration, not browser automation.

Strict red lines:

- Do not automate login to consumer web products.
- Do not ask for, store, share, or replay user cookies, browser profiles, or session tokens.
- Do not treat ChatGPT/Claude/Gemini website subscriptions as backend APIs.
- Do not hide fallback/local generation as verified research.

The product exposes three agent modes:

- **Fast mode**: lowest cost and fastest loop; bounded or disabled research; cheap/local/fallback models first.
- **Research mode**: balanced quality; public-source cascade, source-gap brief, mid-tier model routing, and stronger selective QA.
- **Enterprise mode**: BYOK or customer-owned enterprise providers, stronger final polish, strict audit trail, and human review hooks.

## Accessibility & Inclusion

Target WCAG AA for text contrast, keyboard navigation, focus states, and form semantics. Respect reduced-motion preferences. Chinese and English users should both be able to understand the core workflow, with the current local workflow optimized for Chinese.
