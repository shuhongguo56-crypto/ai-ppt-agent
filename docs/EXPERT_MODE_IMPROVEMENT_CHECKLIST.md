# 专家模式改进清单

更新日期：2026-07-19

本清单把 100 个产品问题的已确认答案转成可执行验收项。默认决策是用户连续选择的方案 1；例外包括：允许全网图片进入候选池、授权不明图片仅可预览、公共素材库自动增长、按质量分与分类动态前 10% 晋级。

## P0：客户交付硬门槛

- [x] PPTX 与 HyperFrames HTML 继续共享唯一 `SlideDeck JSON`。
- [x] 研究模式与企业模式统一启用专家级交付门槛。
- [x] 读取 PNG/JPEG 真实像素，不再信任供应商声明尺寸。
- [x] 普通页视觉资产默认不低于 1920×1080；封面与收束页默认不低于 3840×2160。
- [x] 接入官方 Real-ESRGAN 本地超分适配器；低分辨率素材自动增强后再进入渲染。
- [x] 质检新增 `visual_asset_resolution_quality`，分辨率不达标时阻止正式导出。
- [x] 取消“每页必须全屏背景图”的错误质量规则。
- [x] 质检新增 `pptx_visual_placement_diversity`；8 页稿至少覆盖 6 种视觉重心。
- [x] PPTX 与 HTML 取消统一的全页重蒙版；窗口图、条带图、边缘图默认显示原图。
- [x] 只有真正的全出血叙事页使用局部阅读区，不再给所有图片盖深色层。
- [x] 清除渲染器内残留的乱码清洗字符；中文固定宋体，英文固定 Times New Roman。
- [x] 提升标题、观点句与卡片正文的字号层级，继续由文本适配器防止越界。
- [x] PowerPoint COM 真机打开并导出 PNG，纳入放映兼容性烟测。

## P0：图片、授权与素材库

- [x] 开放授权图片搜索顺序调整为 Openverse → Wikimedia Commons → Wikipedia → Bing。
- [x] 搜不到合适开放授权图片时，再调用 AI 生图。
- [x] 授权不明的网页图片保留为预览候选；研究/企业正式渲染会优先改用 AI 图。
- [x] 资产 sidecar 写入 `licenseStatus` 与 `requiresUserConfirmation`。
- [x] 质检新增 `visual_asset_license_readiness`；未确认的未知授权图片阻止正式交付。
- [x] 建立项目自有构图素材库，覆盖 16 种构图家族、图片几何、视觉重心与局部阅读区。
- [x] 建立可增长的公共图片素材库；仅 AI 生成图与开放授权图可自动晋级。
- [x] 公共素材晋级带隐私硬门、80 分质量底线和分类动态前 10% 阈值。
- [x] 素材库记录哈希、尺寸、来源、授权、归因、标签和质量分，避免重复入库。
- [x] 记录 PPT Master、PPTAgent、Presenton、PptxGenJS、Real-ESRGAN 的许可证与可借鉴原则；不复制第三方成品模板。

## P0：页面规划与版式多样性

- [x] 封面支持右侧主视觉、左侧编辑图、上方建筑感全景等不同视觉重心。
- [x] 目录页、过程页、证据页、系统页、对比页、优先级页和收束页使用不同图片几何。
- [x] 修复编辑式封面“图片和文字都在左侧”的碰撞。
- [x] 修复编辑式分栏页，图片固定在左侧边缘，所有内容转移到右侧安全区。
- [x] 修复证据拼贴页标题与右上图片碰撞。
- [x] 修复数据页标题、数据卡和证据图碰撞。
- [x] 修复对比页中间视觉与两侧卡片碰撞。
- [x] HTML 写入 `data-visual-mode` 与 `data-visual-gravity`，与 PPTX 使用同一构图定义。
- [x] 去掉 HTML 中把图片重新强制放回右侧的遗留覆盖规则。
- [ ] 把每个构图家族继续扩展到 3–5 个可验证变体，并按内容密度选择。
- [ ] 为图表页补充真正可编辑的图表、表格和信息图元件，而不是只使用装饰形状。

## P1：内容与研究能力

- [ ] 把多来源事实核验细分为事实、推断、观点和待确认信息。
- [ ] 为研究模式增加来源覆盖率、逻辑闭合度、反例覆盖和结论可行动性评分。
- [ ] 把文章段落映射到每页大纲依据，保证所有可见内容都能回溯到已确认大纲。
- [ ] 没有上传资料时，生成“检索问题树 → 多源检索 → 证据矩阵 → 大纲决策”。
- [ ] 引用格式支持网页、论文、财报和法规，并在讲稿备注中保留完整来源。
- [ ] 企业模式增加事实冻结日期与过期信息提醒。

## P1：图片理解与检索

- [ ] 增加人脸、品牌、敏感场景和伪文字检测。
- [ ] 根据页面实际裁切比例评估主体安全区，不只检查原图分辨率。
- [ ] 增加图文语义相似度模型，低相关图片自动换图。
- [ ] 支持同一主题下的角色、产品、色彩与镜头连续性。
- [ ] 素材库增加候选检索与复用接口；只有标签与内容目标匹配时才复用。
- [ ] 为全网预览图增加用户授权确认界面与替换按钮。

## P1：自动质检与返修

- [x] 现有质量闭环继续自动阻止未通过稿件导出。
- [x] 分辨率、授权、图片唯一性、构图多样性纳入企业级基线。
- [x] PPTX 与 HyperFrames 最终成品新增可见文本完整性检查；标题或正文以省略号结尾会阻止交付。
- [x] 中英文长标题先按语义边界压缩，再进入同一 SlideDeck/PPTX/HTML 渲染链路。
- [ ] PowerPoint 真机截图进入视觉回归：越界、遮挡、过小字号、低对比、主体裁切、重复重心。
- [ ] 自动返修按失败项定向修改，不重新生成已通过的内容和图片。
- [ ] 为每页输出机器可读的几何占用图，检查文字、图片、卡片和图表之间的碰撞。
- [ ] 增加手机端 HyperFrames 全屏放映、横竖屏切换与触控手势回归。

## P2：前端与商业化

- [ ] 客户前端继续保持单问题、下一步换页，不展示模型路由和后台日志。
- [ ] 快速模式、研究模式、企业模式只展示结果差异、预计时间与消耗，不堆技术解释。
- [ ] 视觉方向改为内容驱动的动态数量，不固定三套；每套使用真实缩略图预览。
- [ ] 用户可锁定页、内容、图片或版式后局部重做。
- [ ] 增加 BYOK、混合路由和人工提示词工作台三种合法低成本架构。
- [ ] 完成认证、订阅、积分账本、对象存储、任务队列和生产数据库。
- [ ] 公共站点增加稳定域名/隧道健康监控和部署回滚。
