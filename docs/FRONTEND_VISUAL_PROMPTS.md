# 客户端视觉提示词资产

## 参考结论

本轮只吸收 MotionSites 的展示方法，不复制其付费模板或提示词：深色作品画廊、超大标题、动态背景、成品优先、短文案与强对比。参考页面：

- https://motionsites.ai/

- https://motionsites.ai/backgrounds

## 前端总提示词

```text
你是一位同时具备 Apple 产品设计、国际演示设计与动态网页导演经验的资深设计负责人。

任务：设计一个面向真实客户的 AI PPT 创作工作台。它不是模型控制台、开发者后台或数据仪表盘。客户来到这里，只关心自己的内容、大纲、视觉风格、页面预览和最终文件。

产品承诺：用户输入主题、文字或上传资料；系统先给出可审阅大纲，用户确认后再生成与内容匹配的视觉方向，随后生成同一份 SlideDeck JSON，并导出可编辑 PPTX 与动态 HTML。

信息层级：
1. 首屏只出现一句结果导向标题、一句短说明、一个明确的“开始创作”按钮，以及一张能体现高级 PPT 质感的动态成品预览。
2. 主流程使用逐题出现的对话式输入；一次只要求用户做一个决定。
3. 进度只使用客户语言：提出需求、确认大纲、选择风格、生成页面、预览下载。
4. API 地址、模型路由、图片供应商、JSON、质检明细和调试信息默认不可见，统一收进右下角“专业设置”。
5. 任何后台能力都先转译成客户收益；默认界面不得解释技术实现。

视觉语言：
- 采用黑色/石墨色作品画廊底色，奶油白正文，冷蓝作为唯一主强调色。
- 首屏像一场高端发布会：超大但克制的排版、明确留白、单个视觉主角、精细景深。
- 用错位叠放的 16:9 幻灯片形成空间感；背景卡片只承担层次，不抢正文。
- 作品卡片接近真实 PPT 比例，圆角克制，边框极轻，光影遵循同一光源。
- 使用编辑设计网格，不使用廉价的 AI 渐变气泡、无意义光斑、满屏玻璃卡片或卡片套卡片。
- 不固定 Apple/McKinsey/Airbnb 三套模板；视觉方向必须由主题、受众、证据密度、情绪和图片策略动态生成。

动效：
- 页面出现使用 280–420ms 的淡入、轻微上移和清晰度恢复。
- PPT 预览支持鼠标拖动产生小幅 3D 倾斜，松手后带惯性并回到稳定位置。
- 背景幻灯片只做缓慢错位漂移；正文绝不持续浮动。
- 所有按钮按下时立即缩小 2–3%，悬停只提升 1–2px。
- 尊重 prefers-reduced-motion；移动端减少透视和背景动画。

响应式：
- 桌面首屏为“结果导向文案 + 动态 PPT 画廊”双栏。
- 平板与手机改成单栏，标题保持完整，按钮不溢出，PPT 预览保持 16:9 关系。
- 移动端隐藏次要状态文字，仅保留项目库、当前步骤和服务状态点。

质量门槛：
- 默认视图中不能出现 Frontend-Slides、HyperFrames、provider、adapter、API chain 等技术名词。
- 不允许横向溢出、正文覆盖图片、标题超出画布、低对比文本或无法键盘访问的交互。
- 空状态也必须像可交付产品，而不是后台占位页。
```

## 动态 PPT 视觉方向提示词

这三段不是固定模板，而是视觉生成器可组合的方向原语。生成时必须先读取 `OutlineDecision`、受众、内容密度、情绪、证据类型和 `image_plan`，再决定是否采用或混合。

### 方向原语 A：Midnight Keynote

```text
为当前大纲生成“深夜发布会”视觉方向：黑曜石舞台、奶油白大字、单一冷蓝强调、电影级全幅图片、极强留白。每页只保留一个视觉主角；标题控制在两行内；证据页使用精准标注和局部聚光，不使用整齐重复的三卡片布局。动效以遮罩揭示、景深推进和轻微视差为主。所有页面构图必须因内容角色而不同。
```

### 方向原语 B：Editorial Lens

```text
为当前大纲生成“编辑电影感”视觉方向：把内容当成一篇高端杂志专题和纪录片分镜。使用非对称网格、图像裁切、章节页留白、引文与证据的尺度对比。中文字体遵循宋体体系，英文字体遵循 Times New Roman 体系；标题、正文、注释形成明确字号阶梯。图片必须解释当前论点，不得仅作装饰。转场采用章节切片、图片焦点迁移与文字逐层显现。
```

### 方向原语 C：Spatial Evidence

```text
为当前大纲生成“空间证据”视觉方向：把数据、流程、比较和结论组织成具有纵深的证据场。以严格网格和少量悬浮层建立景深，避免仪表盘感。数据页使用可编辑图表或图解，论证页使用因果链、对照关系或焦点图片。每页的主信息必须在三秒内被识别；次级信息沿阅读路径展开。动效服务论证顺序，不做无意义循环动画。
```

## 负面提示词

```text
禁止：固定三模板、同一图片跨页复用、每页相同分区、卡片矩阵滥用、文字堆叠、正文盖住图片、背景图带不可控文字、标题超过三行、超出 16:9 安全区、随机装饰图、廉价霓虹、加密货币式仪表盘、无意义渐变气泡、玻璃卡片套玻璃卡片、为了动效而动效、暴露模型或 API 调试信息。
```

## 客户可见性约束

| 默认可见 | 仅在专业设置可见 |
| --- | --- |
| 主题、资料、受众、语言 | API 地址、模型提供商与路由 |
| 大纲审阅与修改 | 图片适配器与供应商顺序 |
| 内容驱动的视觉方向 | SlideDeck JSON 与调试日志 |
| 逐页预览、放映、下载 | 质检明细、错误追踪与成本策略 |
